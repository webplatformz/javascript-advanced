Function

A function is a process which takes some input, called arguments, and produces some output called a return value

fn(in) => out

Purpose

(mapping) A function maps input values to output values.​

(procedure) A function may perform a sequence of steps.

(i/o) A function may communicate with other parts of the system.

High Order Function

A function which takes a function as an argument, returns a function, or both.

A function is pure if …

… given the same input, will always return the same output.​

… produces no side effects.​

A pure function …

… does not rely on any external mutable state.​

… does not mutate external state.​

A dead giveaway that a function is impure is if it makes sense to call it without using its return value. For pure functions, that’s a noop.

Properties & Aspects (1)

deterministic

predictable

completely independent of outside state​

immune to bugs related to shared mutable state​

​

Properties & Aspects (2)

great candidates for parallel processing

easy to move around, refactor and reorganize

flexible and adaptable to future changes​

very easy + safe to reuse (no state, no side-effects)​

​

Properties & Aspects (3)

referentially transparent

= any call to a pure function can be replaced with its resulting value without changing the meaning of the program​

possibilities vs consequences

Problems with mutable state:

Tracking mutations in code is tiresome

Code is hard(er) to read and error prone

Code is hard(er) to test and debug

It's most likely not multi-thread safe

Tracking changes and unwanted side-effects in objects is difficult and very expensive

Trivial example

let animals = [ 'hamster', 'cat', 'mouse', 'dog' ];
let sortedAnimals = animals.sort();
console.log(animals === sortedAnimals); // false

Not so trivial example

let sum = 0;
for (let index = 0; index < data.length; index++) {
  sum += data[index];

Objects, Arrays - Object.assign()

let animal = {
    name: "Homer",
    age: 2,
    type: "hamster",
    legs: 4
};
let shallowCopy = Object.assign({}, animal);
console.log(animal === shallowCopy); // false

Objects, Arrays - Destructors

let animal = {
    name: "Homer",
    age: 2,
    type: "hamster",
    legs: 4
};
let shallowCopy = { ...animal };
console.log(animal === shallowCopy); // false

Be aware

{ ...oldObj }, Object.assign(), $.extend(), ... are flat shallow copies

Not suitable for more complex (deep) objects

Advantage - Disadvantage

- Very tiny performance overhead

+ concurrency safe (multi-threading, asynchronicity, ...)

+ No expensive state tracking Object.observe(obj, fn)

+ different/functional application design

+ It's a simple pattern. You can assume everything immutable
